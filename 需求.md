## 一、项目业务需求（wordMemory）

开发一款浏览器端原生单词记忆网站：
1. 需求分析：

2. 单词记忆模块击单词记忆模块，选择记忆的单词数，在单词本里随机不重复选择生成单词，展示在页面，每个单词后有两个按钮：会/不会，点击不会，显示单词释义并将单词添加到生词本

3. 单词抽查模块：随机选择单词（尽量选择生词本里的单词），文本框里输入释义，点击确定，如果正确，则显示正确（近义词也算正确），否则显示错误，错误次数加1，错误时间记录为当前时间，并显示当前单词正确意思

4. 生词本模块，记录曾经错误或者不会的单词

5. 生词本排序：点击生词本模块，可根据错误次数、错误时间（可重复）进行排序
6. 清空生词本：点击清空生词本，将生词本数据清空

7. 本地持久化：生词本、单词以json格式存储在浏览器本地不同的文件夹，页面刷新后数据不会丢失

8. 可拓展性：单词文件夹中每加入一个新的json文件，都在前端做映射，将单词模块的json文件映射为子模块



## 二、分阶段流程
### 阶段1：SDD 软件详细设计文档
| 产出物          | 验收标准（必须包含）                                         |
| :-------------- | :----------------------------------------------------------- |
| 完整SDD设计文档 | 项目使用框架（或者原生语言）、项目整体架构、页面结构拆分、存储方案、入参出参说明、边界异常场景梳理、文件目录结构 |

### 阶段2：Spec 标准化需求规格

基于SDD生成可编码的标准化需求规格

| 产出物         | 验收标准                                                     |
| :------------- | :----------------------------------------------------------- |
| 结构化Spec文档 | 逐条定义4项功能的精准约束、正常用例、边界用例（空输入、重复任务、刷新丢失、删除异常）、代码强制规范，无模糊口语化需求 |

### 阶段3：Write-plans 编写分步开发计划

基于Spec生成可执行分步开发计划

| 产出物       | 验收标准                                                     |
| :----------- | :----------------------------------------------------------- |
| 分步开发Plan | 1. 拆分页面搭建、新增、完成、删除、本地存储5个独立开发步骤；2. 标注每一步开发风险点；3. 计划与前置SDD、Spec完全对齐；4. 明确每一步代码交付要求 |

### 阶段4：Execute-plan 执行开发计划

严格按照已生成plan分步执行编码

| 产出物                 | 验收标准                                                     |
| :--------------------- | :----------------------------------------------------------- |
| 可运行TodoList完整源码 | 1. 严格按计划分步编码，无跳步开发；2. 所有核心功能全部可用；3. 控制台无报错；4. 代码结构和前期设计一致 |

### 阶段5：TDD 测试驱动开发

| 产出物          | 验收标准                                                     |
| :-------------- | :----------------------------------------------------------- |
| 全套TDD测试用例 | 覆盖上述所有场景，包含正常用例+边界异常用例；<br>留存先红（测试失败）后绿（测试通过）完整迭代日志 |
